<div class="post-chapter">1. 진행 방향</div><br>
01편에서 정리했던 아쉬운 점은 다음과 같다.<br>
<br>
🔸와이어 프레임만 존재 : 디자인 자료 부족<br>
🔸화면 설계서의 상세 설명 부족<br>
🔸디자인 문제 : UI 너무 큼, 디자인 별로<br>
🔸부족한 수정 정보 : 로그 없음, 등록/수정 내역 없음.<br>
<br>
그리고 개발 기한은 없다지만 모든 걸 한번에 개발하기는 힘들 것 같아서<br>
1차적으로 개발할 기능만 골라서 개발하기로 했다.<br>
<br>
위 내용들을 종합해서 결정한 최종 진행 방향은:<br>
<br>
🔹사용자/관리자 화면을 분리한다.<br>
🔹사용자 화면은 PC/모바일 양쪽으로 사용이 가능하게 만든다.<br>
🔹회원 관리 기능은 제외하고, 모든 사용자가 사용할 수 있는 메뉴부터 만든다.<br>
   => 메인화면, 칵테일 정보, 칵테일 계산기 등<br>
🔹관리자 화면은 PC 기준으로 만들고, 모바일은 고려하지 않는다.<br>
🔹관리자 화면은 관리자 로그인을 통해 진입하고, 데이터 관리 메뉴를 만든다.<br>
   => 칵테일 정보 관리, 재료 정보 관리, 코드 관리 등<br>
🔹관리자 로그인 시도 내역과 데이터 수정 내역을 별도 로그로 남긴다.<br>
🔹화면설계는 와이어프레임에 상세하게 작성하고, 피그마로 별도로 디자인을 만든다.<br>
🔹문서는 기능명세, 화면설계서, 디자인 시안을 작성한다.<br>

🔹FE : React + Typescript<br>
<br>
React의 컴포넌트 구조가 깔끔하고 좋았던 것 같아서 프론트는 React로 결정했다.<br>
React에 Typescript가 붙었을때 구조가 더 명확해지는 것도 좋았어서 Typescript도 쓸 예정이다.<br>
<br>
React니까 상태 관리가 필요할수 있을텐데, <br>
Redux 정도만 사용하거나 최대한 사용하지 않는 방향으로 작업하려고 한다.<br>
<br>
라이브러리를 쓰면 편하고 좋긴 하지만.. 순정 코딩을 할때의 그 강력한 느낌이 더 좋다.<br>
<br>
<img src="{{ "/assets/img/nectar/00012/nectar_00012_003.png" }}" style="width: 10rem;"/><br>

<div class="post-chapter">3. 모듈 분리</div><br>
관리자 웹은 관리자 몇명만 사용하지만 사용자 웹은 여러 명을 대상으로 하기 때문에 <br>
사용자 웹의 스케일 관리를 감안해서 관리자 웹 / 사용자 웹 모듈은 별도로 개발 예정이다.<br>
같은 이유로 Admin API / Client API 도 분리하고,<br>
DB는 접근 권한을 통해서 각 모듈이 다르게 사용할 수 있도록 설정할 것이다.<br>
